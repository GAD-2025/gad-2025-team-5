## 1. 라우팅 구조 (`App.js`)

앱의 전체적인 페이지 이동은 `react-router-dom`을 통해 관리됩니다. 주요 경로는 다음과 같습니다.

-   `/`: **스플래시 스크린** (`SplashScreen.jsx`)
-   `/login`: **로그인 페이지** (`LoginPage.jsx`)
-   `/signup`: **회원가입 페이지** (`RegisterPage.jsx`)
-   `/onboarding`: **온보딩 (관심사 선택) 페이지** (`Onboarding.js`)
-   `/home`: **메인 홈 페이지** (`Home.js`)
-   `/list/*`: **일반 목록 페이지** (`List.js`)
-   `/category/:categoryName`: **카테고리별 도서 목록 페이지** (`CategoryPage.jsx`)
-   `/books/:id`: **도서 상세 페이지** (`BookDetail.jsx`)
-   `/payment`: **결제 페이지** (`PaymentPage.js`)
-   `/register`: **상품 등록 1단계 페이지** (`Register.js`)
-   `/register2`: **상품 등록 2단계 페이지** (`RegisterPage2.js`)
-   `/registering`: **상품 등록 중 로딩 페이지** (`RegisteringPage.jsx`)
-   `/community`: **커뮤니티 메인 페이지** (`CommunityPage.jsx`)
-   `/community/:id`: **커뮤니티 게시글 상세 페이지** (`CommunityPostDetail.jsx`)
-   `/chat`: **채팅 목록 페이지** (`ConversationListPage.jsx`)
-   `/chat/:id`: **개별 채팅 페이지** (`ChatPage.jsx`)
-   `/mypage`: **마이페이지** (`MyPage.jsx`)

## 2. 페이지별 상세 설명

### 2.1. `SplashScreen.jsx`

-   **경로**: `/`
-   **설명**: 앱 실행 시 가장 먼저 나타나는 스플래시 화면입니다. 3초 동안 앱의 로고("책담")를 보여준 뒤, 자동으로 로그인 페이지(`/login`)로 이동합니다.
-   **CTA 및 기능**:
    -   사용자와의 상호작용이나 CTA는 없습니다.

### 2.2. `LoginPage.jsx`

-   **경로**: `/login`
-   **설명**: 사용자가 이메일과 비밀번호를 사용하여 앱에 로그인하는 페이지입니다.
-   **CTA 및 기능**:
    -   **로그인 버튼**: 사용자가 입력한 정보로 `/api/auth/login`에 로그인을 요청합니다.
        -   성공 시: 신규 사용자는 온보딩 페이지(`/onboarding`)로, 기존 사용자는 홈 페이지(`/home`)로 이동합니다.
        -   실패 시: 에러 메시지를 포함한 팝업 모달이 표시됩니다.
    -   **비밀번호를 잊으셨나요? 링크**: 현재는 콘솔에 메시지만 출력하며, 기능은 구현되어 있지 않습니다.
    -   **회원가입 버튼**: 회원가입 페이지(`/signup`)로 이동합니다.

### 2.3. `RegisterPage.jsx`

-   **경로**: `/signup`
-   **설명**: 신규 사용자 등록을 위한 첫 단계 페이지입니다. 닉네임, 이메일, 비밀번호를 입력하고 약관에 동의합니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 로그인 페이지(`/login`)로 돌아갑니다.
    -   **프로필 사진 등록**: 클릭 시 파일 탐색기를 열어 사용자가 프로필 사진을 선택하고 업로드할 수 있습니다. (선택 사항)
    -   **중복 확인 버튼**: 닉네임 중복 여부를 서버에 확인합니다. 이 확인 절차를 통과해야만 회원가입을 진행할 수 있습니다.
    -   **약관 동의**: '전체 동의'를 포함한 필수 약관들을 체크해야 합니다.
    -   **가입하기 버튼 (Primary CTA)**: 모든 필수 정보 입력 및 약관 동의가 완료되면 활성화됩니다.
        -   클릭 시, 입력된 사용자 정보(닉네임, 이메일, 비밀번호)를 `localStorage`에 임시 저장하고, 다음 단계인 온보딩 페이지(`/onboarding`)로 이동합니다.

### 2.4. `Onboarding.js`

-   **경로**: `/onboarding`
-   **설명**: 신규 사용자의 취향을 파악하기 위한 2단계의 온보딩 페이지입니다.
-   **CTA 및 기능**:
    -   **1단계 (장르 선택)**:
        -   **장르 버튼**: 여러 개의 장르 중 관심 있는 장르를 복수 선택할 수 있습니다.
        -   **선택 완료 버튼**: 1개 이상의 장르를 선택하면 활성화되며, 2단계로 넘어갑니다.
    -   **2단계 (도서 선택)**:
        -   **뒤로가기 버튼**: 이전 단계인 장르 선택 화면으로 돌아갑니다.
        -   **도서 검색창**: 사용자가 좋아하는 책을 검색할 수 있습니다.
        -   **도서 카드**: 검색된 도서 목록에서 3개 이상의 책을 선택해야 합니다.
        -   **완료 버튼 (Primary CTA)**: 3개 이상의 책을 선택하면 활성화됩니다.
            -   클릭 시, `localStorage`에 저장된 사용자 정보와 온보딩에서 선택한 장르 및 도서 정보를 종합하여 `/api/auth/register-complete` 엔드포인트로 전송하여 최종적으로 회원가입을 완료합니다.
            -   성공 시, 인증 토큰을 저장하고 홈 페이지(`/home`)로 이동합니다.

### 2.5. `Home.js`

-   **경로**: `/home`
-   **설명**: 로그인 후 가장 먼저 보게 되는 메인 페이지입니다. 사용자 맞춤 추천 도서, 실시간 매물, 커뮤니티 인기글 등 다양한 정보를 제공합니다.
-   **CTA 및 기능**:
    -   **상단 헤더 아이콘**:
        -   **검색 아이콘**: 클릭 시 `SearchModal` (검색 모달)을 엽니다.
        -   **알림 아이콘**: 현재 기능이 구현되어 있지 않습니다.
        -   **메뉴(햄버거) 아이콘**: 클릭 시 전체 카테고리를 볼 수 있는 사이드 메뉴를 엽니다.
    -   **카테고리 메뉴**: 하위 카테고리 클릭 시 해당 카테고리 페이지(`/category/:categoryName`)로 이동합니다.
    -   **추천 도서 섹션**:
        -   `오늘의 추천`, `인기`, `내 취향 맞춤` 탭을 통해 도서 목록을 필터링할 수 있습니다.
        -   **전체 보기 링크**: 추천 도서 전체 목록 페이지(`/list/recommendations`)로 이동합니다.
    -   **실시간 매물 도서 섹션**:
        -   `새 매물`, `가격 인하` 버튼으로 목록을 필터링합니다.
        -   **전체 보기 링크**: 실시간 매물 전체 목록 페이지(`/list/realtime`)로 이동합니다.
    -   **도서 카드**: 클릭 시 해당 도서의 상세 페이지(`/books/:id`)로 이동합니다.
    -   **커뮤니티 인기글**: 클릭 시 해당 게시물의 상세 페이지(`/community/:id`)로 이동합니다.

### 2.6. `CategoryPage.jsx`

-   **경로**: `/category/:categoryName`
-   **설명**: 특정 카테고리에 속한 모든 책을 보여주는 페이지입니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 홈 페이지(`/home`)로 돌아갑니다.
    -   **도서 카드 (Primary CTA)**: 페이지에 표시된 책 카드를 클릭하면 해당 책의 상세 페이지(`/books/:id`)로 이동합니다.
    -   **로딩/빈 상태**: 데이터를 불러오는 동안 로딩 메시지를, 해당 카테고리에 책이 없으면 빈 상태 메시지를 표시합니다.

### 2.7. `List.js`

-   **경로**: `/list/*` (예: `/list/recommendations`)
-   **설명**: 홈 페이지의 '전체 보기' 링크를 통해 진입하는 일반적인 도서 목록 페이지입니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 홈 페이지(`/home`)로 돌아갑니다.
    -   **도서 카드 (Primary CTA)**: 도서 카드를 클릭하면 상세 페이지(`/books/:id`)로 이동합니다.
    -   **하단 네비게이션 바**: 홈, 커뮤니티, 등록, 채팅, 마이페이지로 이동할 수 있는 링크를 제공합니다. (현재 일부만 `Link`로 활성화)

### 2.8. `BookDetail.jsx`

-   **경로**: `/books/:id`
-   **설명**: 개별 도서의 모든 상세 정보를 보여주는 페이지입니다. 책 이미지, 가격, 설명, 판매자 정보 등을 포함합니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 이전 페이지로 돌아갑니다.
    -   **좋아요(하트) 버튼 (Primary CTA)**: 책에 대한 '좋아요' 상태를 토글합니다. 이 상태는 서버에 저장되어 모든 사용자가 공유합니다.
    -   **하단 구매 바**:
        -   **채팅하기 버튼 (Primary CTA)**: 판매자와의 1:1 채팅을 시작합니다. 기존 채팅이 있으면 해당 채팅방으로, 없으면 새로운 채팅방을 생성하여 `/chat/:chatId`로 이동합니다.
        -   **구매하기 버튼 (Primary CTA)**: 해당 도서 정보를 가지고 결제 페이지(`/payment`)로 이동합니다.

### 2.9. `PaymentPage.js`

-   **경로**: `/payment`
-   **설명**: 도서 구매의 마지막 단계인 주문/결제 페이지입니다. 주문 상품, 배송지 정보, 결제 수단을 보여줍니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 이전 페이지(도서 상세)로 돌아갑니다.
    -   **배송지 정보**: 현재는 하드코딩된 기본값이 표시되며, 수정 기능은 없습니다.
    -   **결제 수단 버튼**: 신용카드, 무통장입금 등의 결제 방법을 선택할 수 있습니다. 클릭 시 시각적으로 활성화되지만, 실제 결제 연동은 되어있지 않습니다.
    -   **결제하기 버튼 (Primary CTA)**: 클릭 시 '결제가 완료되었습니다'라는 `alert`를 띄우고, 이전 페이지로 돌아갑니다.

### 2.10. `Register.js` (상품 등록 1단계)

-   **경로**: `/register`
-   **설명**: 판매할 중고 도서를 등록하는 첫 번째 단계입니다. 바코드 스캔, 사진 등록, 책 정보(제목, 작가, 설명 등), 가격, 배송 옵션을 입력합니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 홈 페이지(`/home`)로 돌아갑니다.
    -   **바코드 스캔하기 버튼**: 카메라를 이용한 바코드 스캐너를 엽니다. 스캔 성공 시 ISBN을 추출하여 책 정보를 자동으로 채워줍니다.
    -   **ISBN 직접 입력/검색**: 수동으로 ISBN을 입력하여 책을 검색할 수 있습니다.
    -   **사진 등록**: 최대 5장의 책 사진을 등록할 수 있습니다.
    -   **가격 제안 받기 토글**: 판매자가 가격 제안을 받을지 여부를 선택합니다.
    -   **다음 버튼 (Primary CTA)**: 모든 필수 항목이 채워지면 활성화됩니다. 입력된 데이터를 `RegistrationContext`에 저장하고, 등록 2단계 페이지(`/register2`)로 이동합니다.

### 2.11. `RegisterPage2.js` (상품 등록 2단계)

-   **경로**: `/register2`
-   **설명**: 상품 등록의 마지막 단계로, 책의 상태 등급(S급, A급 등)을 선택합니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 이전 단계인 `/register` 페이지로 돌아갑니다.
    -   **상태 등급 선택**: S급부터 D급까지 책의 상태를 선택해야 합니다.
    -   **등록하기 버튼 (Primary CTA)**: 등급이 선택되면 활성화됩니다.
        -   클릭 시, 로딩 페이지(`/registering`)로 잠시 이동합니다.
        -   1단계에서 `RegistrationContext`에 저장된 데이터와 현재 선택된 등급 정보를 합쳐 `FormData`를 생성하고, `/api/books`로 POST 요청을 보내 최종적으로 상품을 등록합니다.
        -   등록 성공 시, 새로 등록된 책의 상세 페이지(`/books/:id`)로 이동합니다.

### 2.12. `RegisteringPage.jsx` (로딩)

-   **경로**: `/registering`
-   **설명**: `RegisterPage2`에서 '등록하기' 버튼을 누른 후, 서버에 데이터가 전송되고 처리되는 동안 잠시 표시되는 로딩 페이지입니다.
-   **CTA 및 기능**:
    -   사용자와의 상호작용이나 CTA는 없습니다. "등록 중이에요!" 메시지를 표시하며 대기하도록 안내합니다.

### 2.13. `CommunityPage.jsx`

-   **경로**: `/community`
-   **설명**: 다양한 주제의 커뮤니티 게시글 목록을 보여주는 페이지입니다. 현재 게시글 데이터는 프론트엔드에 하드코딩되어 있습니다.
-   **CTA 및 기능**:
    -   **필터 탭 (`전체`, `인기글`, `내 글`)**: 클릭 시 해당 카테고리의 글들만 필터링하여 보여줍니다.
    -   **게시글 카드 (Primary CTA)**: 각 게시글은 카드로 표시되며, 클릭 시 해당 게시글의 상세 페이지(`/community/:id`)로 이동합니다.
    -   **좋아요(하트) 버튼**: 게시글의 '좋아요' 수를 토글합니다. (현재는 로컬 상태만 변경)

### 2.14. `CommunityPostDetail.jsx`

-   **경로**: `/community/:id`
-   **설명**: 커뮤니티 게시글 하나의 전체 내용과 댓글을 보여주는 상세 페이지입니다.
-   **CTA 및 기능**:
    -   **뒤로가기 버튼**: 이전 페이지(커뮤니티 목록)로 돌아갑니다.
    -   **좋아요(하트) 버튼**: 게시글의 '좋아요' 상태를 토글합니다. (로컬 상태만 변경)
    -   **댓글 입력창 및 등록 버튼 (Primary CTA)**: 사용자가 새로운 댓글을 작성하고 등록할 수 있습니다. 엔터 키로도 등록 가능합니다. (로컬 상태만 변경)

### 2.15. `ConversationListPage.jsx`

-   **경로**: `/chat`
-   **설명**: 사용자의 모든 1:1 채팅 목록을 보여주는 페이지입니다. 각 채팅방의 마지막 메시지, 시간, 읽지 않은 메시지 수를 표시합니다.
-   **CTA 및 기능**:
